    FAQ

    1.
    Q：WebHook通知重试机制
    A：接收到WebHook通知后，没同步返回“{"code": "200","msg": "Received successfully"}”，则会进行11次重试；
    Retry CountDelay
    11s
    22s
    34s
    48s
    5-710m
    8-101h
    113h
    2.
    Q：3.15 卡交易3DS授权确认、取消，一般需要跟用户交互吗？让用户进行确认？
    A：当用户在做交易时，商家会发起3DS授权交易请求，我们接收到之后会以“8.4 授权3DS通知”的形式通知你们，
    你们拿到通知后，则需要用户进行确认并调用“3.15 卡交易3DS授权确认、取消”接口。(3DS授权不是每笔交易都会触发)
    3.
    Q：虚拟卡不需要商户像实体卡一样，先进行统一购买吧？
    A：不需要，需要创建的时候调用接口即可；
